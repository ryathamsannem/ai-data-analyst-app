## Technical Debt

| Item | Description |
|------|-------------|
| **Monolithic SPA** | ~14k-line `page.tsx` owns upload, all tabs, export, AI — hard to test in isolation |
| **Dual contract models** | `VisualizationContract` (legacy frozen) + `ChartPresentationContract` (platform) coexist |
| **Overview inline charts** | Duplicate Recharts logic vs `ChartRenderer`; export parity requires parallel constants |
| **Backend globals** | Single `df` global; not suitable for multi-tenant production |
| **Intent engine split** | Some routing in `main.py`, some in `intent_engine/` — discovery cost for new contributors |
| **Generated PDF artifacts in repo** | `docs/pdf-validation-screenshots/` regenerated by tests; clutters git status |

## Recommended Next Priorities

1. **Manual 6-surface parity matrix** — 4-region V-Bar, 7+ category H-Bar, line trend, scatter, donut composition, histogram; document screenshots per release.
2. **Browser export smoke test** — Playwright script: load dataset → export Overview PNG, Charts PNG, PDF; assert non-blank image dimensions.
3. **H-Bar axis parity** — narrow pass using existing `AxisPresentationPlan` infrastructure.
4. **Backend session isolation** — replace global `df` before any production multi-user deployment.
5. **Avoid broad refactors** — extract helpers only when fixing a specific bug; preserve AGENTS.md baseline.

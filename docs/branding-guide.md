# Branding guide (V1)

Global branding is controlled by a single config file — no database, auth changes, or per-customer overrides in V1.

**Source of truth:** [`frontend/lib/branding-config.ts`](../frontend/lib/branding-config.ts)

---

## Quick reference

| Field | Purpose |
|-------|---------|
| `appName` | Full product name (sidebar, PDF header default, browser title) |
| `shortName` | Compact name + initials fallback when no logo |
| `supportEmail` | Shown in PDF footer |
| `primaryColor` | Primary brand color (CSS `--primary`, PDF accent default) |
| `secondaryColor` | Accent color (CSS `--accent`, buttons/chips) |
| `pdfFooterText` | Optional override for PDF center footer line |
| `exportFilePrefix` | PDF download filename prefix |
| `logoUrl` | Optional logo image URL |
| `version` | Optional sidebar badge (e.g. `V1`) |

---

## Change the app name

Edit `BRANDING.appName` and `BRANDING.shortName` in `branding-config.ts`:

```ts
export const BRANDING: BrandingConfig = {
  appName: "Acme Analytics",
  shortName: "Acme",
  // ...
};
```

**Updates automatically in:**

- Browser tab title + meta description (`app/layout.tsx`)
- Sidebar title + version badge area
- Mobile header subtitle
- Profile initials fallback in header
- PDF cover/header company name (when Export tab company field is empty)
- Upgrade modal copy
- PDF footer: `Generated by {appName}`

Restart the dev server after editing.

---

## Change brand colors

Edit `primaryColor` and `secondaryColor` (hex):

```ts
primaryColor: "#0f766e",
secondaryColor: "#14b8a6",
```

`BrandingStyles` applies these to CSS variables at runtime:

- `--primary` ← `primaryColor`
- `--accent` ← `secondaryColor`

PDF exports use `primaryColor` as the default accent when the Export tab accent field is empty.

For deeper theme tweaks (backgrounds, borders), edit [`frontend/app/globals.css`](../frontend/app/globals.css) — keep brand colors in `branding-config.ts` first.

---

## Replace the logo

Set a public URL or path served by Next.js:

```ts
logoUrl: "/brand/logo.svg",
```

Place the asset under `frontend/public/brand/logo.svg`.

When `logoUrl` is set, the sidebar and header use the image. When unset, initials from `shortName` appear in a branded chip (e.g. **AN** for “Analyst”).

**Tips:**

- Prefer SVG or PNG with transparent background
- Keep logos roughly square; they render in a 36×36px shell

## PDF export filename

Downloads use:

```text
{exportFilePrefix}-analytics-report.pdf
```

Example with default prefix:

```text
analytics-analytics-report.pdf
```

Change `exportFilePrefix`:

```ts
exportFilePrefix: "acme",
// → acme-analytics-report.pdf
```

---

## PDF footer

Default footer (center):

```text
Generated by {appName}
Support: {supportEmail}
```

Optional custom center line:

```ts
pdfFooterText: "Confidential — internal use only",
```

When `pdfFooterText` is non-empty, it replaces the “Generated by …” line. The support line still renders below.

---

## Export tab overrides (unchanged)

The Export tab still allows per-session **company name**, **tagline**, and **accent** for PDFs. Those fields override PDF header company/accent only; global `appName` remains the fallback when company name is blank.

Export tab values are stored in localStorage separately from global branding.

## Where branding is applied

| Location | Config fields |
|----------|----------------|
| `app/layout.tsx` | `appName`, description |
| `components/app-shell/app-sidebar.tsx` | `appName`, `version`, `logoUrl` / initials |
| `components/app-shell/app-header.tsx` | `appName`, initials, `logoUrl` |
| `app/pdf-report.ts` | filename, footer, default company/accent |
| `app/components/upgrade-plan-modal.tsx` | `appName` |
| `components/branding/branding-styles.tsx` | colors |

# PDF Export — Stable Baseline (Phase 2)

**Recovery snapshot:** May 2026 — after Premium Export Polish Parts 1–3 and final spacing/footer/appendix/table polish.

**Do not redesign** export architecture or pagination without explicit approval. This document describes **current stable behavior**.

---

## 1. Architecture overview

| Piece | Location |
|-------|----------|
| Export UI | `frontend/app/page.tsx` — `activeTab === "export"` |
| PDF engine | `frontend/app/pdf-report.ts` — `runExecutivePdfExport()` |
| Style tokens | `frontend/lib/pdf-enterprise-style.ts` |
| Date normalization | `frontend/lib/pdf-date-format.ts` |
| Raw vs display metrics | `frontend/lib/metric-value-format.ts` |
| Branding storage | `localStorage` key `ai-data-analyst-report-branding-v1` |

**Single entry point:** `runExecutivePdfExport(rawInput)` — used by Export tab and AI Insights export actions (same function, different `includes` / payload).

**Print theme:** PDF always uses **print-safe light palette** via `buildPdfExportTheme()` — independent of app dark/light UI (intentional for readability).

---

## 2. Export flows

### A) Export tab

1. User configures toggles (KPIs, AI insight, chart, preview, data quality, conversation context, technical appendix).
2. `downloadReport()` → `downloadReportImplRef` builds `ExecutivePdfExportInput`.
3. `validateExportMatchesContract` (when applicable) gates chart/insight alignment.
4. `runExecutivePdfExport()` generates A4 PDF and triggers browser download.

### B) AI Insights export

- Same `runExecutivePdfExport()` with insight-focused payload (question, answer sections, ranked signals, capture element from insight chart ref).
- Gated by `insightChartMatchesCurrentQuestion` / export validation on the UI side before call.

## 3. Chart capture (unchanged logic)

| Step | Behavior |
|------|----------|
| Capture DOM | Off-screen refs: `chartCaptureSessionRef` (Charts, 860px), `chartCaptureInsightRef` (Insights plan widths) |
| Primary path | SVG clone + **Canvg** → PNG (`renderChartSvgToPng`) |
| Fallback | **html2canvas** on plot root with explicit dimensions |
| Embed | `computePdfChartEmbedDimensions()` — proportional scale, aspect clamps, centered frame |
| Scale | `PDF_CHART_CAPTURE_SCALE` (2.5) for raster sharpness |

**Do not change** capture selectors or Recharts DOM contract without regression testing.

---

## 4. PDF section order (stable)

1. Cover panel (title, company, tagline, generated date, source file, mapping confidence)
2. Executive snapshot panel (dataset profile, optional mini-KPIs, dominant insight, low-data note if &lt;15 rows)
3. Executive summary (partitioned: scope, question, main takeaway callout, metrics, evidence, chart highlights, confidence chip)
4. KPI cards (2-column grid, equal row heights)
5. AI insight (question, highlighted signals, structured sections or bullets)
6. Optional: AI conversation context
7. Visualization (chart header panel, chart image, analysis context lines, executive insights panel)
8. Optional: Data preview (native jsPDF table — **not** screenshot)
9. Optional: Data quality summary
10. Optional: Technical appendix (metadata grid, source/notes panels, thumbnail list table, chart spec grid, series sample table)
11. Running header/footer on **every** page (`pdfDrawEnterpriseRunningChrome`)

---

## 5. Pagination

| Mechanism | Role |
|-----------|------|
| `ensurePageSpace(h)` | Adds page when `y + h` exceeds `footerY - pageSafe` |
| `breakBeforeMajorSection(reserve)` | New page if insufficient room before large sections |
| `suppressRowPageBreaks` | Monolithic tables (appendix spec/series, small metadata tables) |
| Preview table | Repeats header row on continuation pages |
| KPI rows | `ensurePageSpace` per row — pairs stay together |

**Do not** change `footerY`, `contentTop0`, or page-break thresholds without visual regression.

## 7. Responsive scaling (PDF layout)

| Concern | Behavior |
|---------|----------|
| Chart height | Capped by remaining page space minus reserved insights block |
| Wide preview | Max 7 columns; column width cap ~30% content width |
| Long text | `splitTextToSize`, ellipsis in cells |
| Viz whitespace | Tighter gap when insights panel follows chart (`chartAfterWithInsights`) |

---

## 8. Known stable behaviors

- ISO dates normalized in narrative and tables (`pdf-date-format.ts`).
- No duplicate “highest highest” ranking phrases (`polishPdfBusinessCopy`, ranking helpers).
- Appendix series uses **raw** metric values (not axis percent heuristics).
- Horizontal bar charts remain horizontal in export semantics.
- Insight–chart gates respected before export from Insights.

## 9. Current limitations

| Limitation | Notes |
|------------|--------|
| Preview cap | 10 rows × 7 columns max |
| Thumbnails | Up to 8 session charts in appendix list |
| PDF theme | Light/print only (not app dark theme) |
| `page.tsx` size | Monolithic SPA — export logic embedded |
| html2canvas fallback | Browser-dependent quality |
| Multi-page table frames | Outer rounded rect per preview segment only |
